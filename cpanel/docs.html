XD SMS Gateway API

A full REST + WebSocket API for sending SMS messages through your Android phone. Build integrations, trigger messages from any backend, and monitor delivery in real time.

Base URL

https://api.sms.luffyxd.store

Response format

All responses are JSON. Errors include an error field.

Auth: Bearer token

Dashboard + user actions. Obtained via login. Header: Authorization: Bearer <token>

Auth: API key

For integrations and Android app. Header: X-Api-Key: <key>

Authentication

XD SMS Gateway uses two authentication methods depending on the caller:

MethodHeaderUsed for
Bearer token (JWT)Authorization: Bearer <token>Dashboard, user-level operations
API keyX-Api-Key: <key>Sending SMS, Android app polling

Get your bearer token by calling POST /api/auth/login. Create API keys in the dashboard under API Keys.

Error handling

All errors return a JSON body with an error string and appropriate HTTP status.

// Example error response (HTTP 401) { "error": "Invalid token" } // Example validation error (HTTP 400) { "error": "Phone number and message required" }
StatusMeaning
200Success
201Created
400Bad request — missing or invalid parameters
401Unauthorized — invalid or missing token/key
403Forbidden — IP not allowed
404Not found
500Server error
503Gateway paused

Testing guide

Follow this flow to test the full SMS pipeline end-to-end without an Android device.

Step 1 — Register & get a token

Call POST /api/auth/register with name, email, password. Save the returned token.

Step 2 — Create an API key

Call POST /api/apikey/create with Bearer token. Save the full key shown once.

Step 3 — Queue an SMS

Call POST /api/sms/send with X-Api-Key. Note the request_id.

Step 4 — Simulate the Android app polling

Call GET /api/get with X-Api-Key. You'll receive the queued message.

Step 5 — Report delivery

Call POST /api/done with the id and status: "sent" to mark it complete.

Step 6 — Check history

Call GET /api/sms/status/<request_id> to confirm the status is completed.

curl example — full test flow
# 1. Register curl -X POST https://api.sms.luffyxd.store/api/auth/register \ -H "Content-Type: application/json" \ -d '{"name":"Test User","email":"test@example.com","password":"secret123"}' # 2. Login (save the token) TOKEN=$(curl -s -X POST https://api.sms.luffyxd.store/api/auth/login \ -H "Content-Type: application/json" \ -d '{"email":"test@example.com","password":"secret123"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['token'])") # 3. Create API key (save the key) APIKEY=$(curl -s -X POST https://api.sms.luffyxd.store/api/apikey/create \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{"name":"Test Key"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['key'])") # 4. Queue an SMS curl -X POST https://api.sms.luffyxd.store/api/sms/send \ -H "X-Api-Key: $APIKEY" \ -H "Content-Type: application/json" \ -d '{"phone_number":"+9876543210","message":"Hello from API!"}' # 5. Poll (simulate Android app) curl -H "X-Api-Key: $APIKEY" https://api.sms.luffyxd.store/api/get # 6. Report delivery (use id from step 5) curl -X POST https://api.sms.luffyxd.store/api/done \ -H "X-Api-Key: $APIKEY" \ -H "Content-Type: application/json" \ -d '{"id":"QUEUE_ITEM_ID","status":"sent"}'

Android integration guide

The Android gateway app polls the API for queued messages, sends them via the SIM, and reports delivery back. Here's the full loop:

1. Connect on startup

Call POST /api/device/connect to register the device and report hardware info.

2. Poll for messages

Every few seconds, call GET /api/get. If success: true, a message is waiting.

3. Send via SIM

Use Android's SmsManager.sendTextMessage() with the phone_number and message fields.

4. Report result

Call POST /api/done with the message id and status: "sent" or "failed".

5. Send heartbeat

Call POST /api/device/update with battery, RAM, network info every 30 s.

Poll queue (GET)

Fetch the next pending message for this API key's account. Returns success: false when the queue is empty.

Webhook guide

Webhooks deliver signed POST requests to your URL on every SMS event. Each payload includes X-Webhook-Signature (HMAC-SHA256) and X-Webhook-Event headers.

Verify signature (Node.js)
const crypto = require('crypto'); function verifyWebhook(body, secret, signature) { const expected = 'sha256=' + crypto .createHmac('sha256', secret) .update(body) // raw body string .digest('hex'); return crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(signature)); }
Event types
EventWhen fired
sms.queuedMessage added to queue
sms.completedDevice reported successful delivery
sms.failedDevice reported failure
sms.cancelledMessage cancelled from queue
